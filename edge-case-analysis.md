# エッジケース分析

## 🔍 現在のロジックで発生する可能性のあるエラー

### 1. **HEX+パーセントの二重#問題**
**状況**: HEX+パーセント形式が`#`で始まる場合
```scss
color: #ff0000 50%;
```

**問題**: 
- `extractColorCodes()`が`#ff0000 50%`全体をマッチ
- `hexPercentToRgba()`が`rgba(255, 0, 0, 0.50)`を返す（#なし）
- `rgbaToHex()`が`#ff000080`を返す（#あり）
- **結果**: `##ff000080`（二重#）

**影響度**: 🔴 高（テストで確認済み）

**修正方法**:
```javascript
// extractColorCodes()で#を除外してマッチ
const hexPercentPattern = /(?:#)?([a-f\d]{3}|[a-f\d]{6})\s*[-,\s]\s*(\d+)%/gi;
// または変換時に#を削除
```

### 2. **RGB値の範囲外エラー**
**状況**: 不正なRGB値
```scss
color: rgb(300, -10, 256);
```

**問題**: 
- `rgbaToHex()`が範囲チェックで`null`を返す
- 変換されずに無視される

**影響度**: 🟡 中（正常動作、エラーではない）

**問題**: 
- `rgbaToHex()`が範囲チェックで`null`を返す
- 変換されずに無視される

### 5. **空の選択範囲**

**影響度**: 🟢 低（正常動作）

### 6. **カラーコードが含まれないテキスト**
**状況**: カラーコード以外のテキストを選択
```scss

```

**問題**: 
- `extractColorCodes()`が空配列を返す
- `convertAndReplaceColors()`が`null`を返す
- 警告メッセージが表示される

**影響度**: 🟢 低（正常動作）

### 7. **正規表現のグローバルフラグ問題**
**状況**: `HEX_PERCENT_PATTERN_GLOBAL`を複数回使用
```javascript
const pattern = HEX_PERCENT_PATTERN_GLOBAL;
pattern.exec(text); // 1回目
pattern.exec(text); // 2回目 - lastIndexが保持される
```

**問題**: 
- グローバルフラグの`lastIndex`が保持される
- 同じパターンを再利用すると予期しない動作

### 8. **大量のカラーコード**
**状況**: 数百個のカラーコードを含むファイル
```scss
// 500個のカラーコード
```

**問題**: 
- 位置重複チェックがO(n²)の複雑度
- パフォーマンス低下

**影響度**: 🟡 中（大規模ファイルで遅延）

---

### 9. **特殊文字を含むテキスト**
**状況**: 正規表現の特殊文字
```scss
color: rgb(255, 0, 0); /* コメント内の rgb(100, 200, 300) */
```

**問題**: 
- コメント内のカラーコードも検出される
- 意図しない変換が発生する可能性

**影響度**: 🟡 中（コメント内も変換される）

---

### 10. **改行を含むカラーコード**
**状況**: 複数行にまたがるカラーコード
```scss
color: rgba(
  255,
  0,
  0,
  0.5
);
```

**問題**: 
- 現在の正規表現は改行を考慮していない
- 検出されない

**影響度**: 🟡 中（一部のフォーマットで動作しない）
